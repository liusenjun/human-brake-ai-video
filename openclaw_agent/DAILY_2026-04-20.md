## MiniMax 图片理解调试 - 最终结论

### 问题描述
Louis 发现 OpenClaw 的 `image` 工具调用 MiniMax M2.7 时报错 "Model does not support images"

### 调试过程
1. 最初怀疑是 baseUrl 配置问题（`/anthropic` vs 原生地址）
2. 尝试将 baseUrl 从 `https://api.minimaxi.com/anthropic` 改为 `https://api.minimaxi.com`
3. 同样失败——baseUrl 不是问题

### 关键发现：官方文档确认
MiniMax 官方 Anthropic API 文档明确写道：
- `type="image"` — **不支持** — 暂不支持图像输入
- `messages` — **部分支持** — 支持文本和工具调用，不支持图像和文档输入

文档地址：https://platform.minimaxi.com/docs/api-reference/text-anthropic-api.md

### 结论
- **M2.7 本身没有图片理解能力** — 纯文本模型
- 图片理解是 **Token Plan 的专属 MCP 工具（MiniMax-VL-01）**，不是 M2.7 的功能
- M2.7 的 `input: ["text", "image"]` metadata 在 OpenClaw 配置里是误导性的

### 已完成的 MCP 配置

**OpenClaw 配置更新：**
- MCP server: `minimax-coding-plan-mcp`（uvx 启动）
- API Key: `sk-cp-...`（已脱敏）
- API Host: `https://api.minimaxi.com`
- 图片理解：`MiniMax-VL-01`（通过 MCP `understand_image` 工具）
- 联网搜索：MCP `web_search` 工具（底层是搜搜/腾讯系中文搜索）

**openclaw.json 配置：**

```json
  }

```

### MiniMax Token Plan vs MCP 工具
- Token Plan 用户可使用 MCP 服务（图片理解 + 联网搜索）
- MCP 工具 `understand_image` 使用 VL-01 模型
- MCP 工具 `web_search` 底层是腾讯搜搜（Sogou）的中文搜索索引
- 普通 API（/anthropic 端点）不支持图片理解，这是商业策略不是技术限制

## LTX2 I2V Repo 分析 — venetanji/ltx2-comfy-v915

### Repo 地址
https://github.com/venetanji/ltx2-comfy-v915

### 核心发现
这个 repo 的 I2V **不是用 Wan**，而是 **LTX2 (Lightricks)** 视频模型。
ComfyUI 脚本实现在 `skills/comfyui/scripts/ltx23.py`。

### LTX2 I2V 使用的模型

| 模型类型 | 文件名 | 加载方式 |
|---------|--------|---------|
| **UNet** | `ltx-2.3-22b-dev-Q4_K_M.gguf` | `UnetLoaderGGUF` |
| **CLIP 1** | `gemma_3_12B_it_fp8_e4m3fn.safetensors` | `DualCLIPLoader` (type=ltxv) |
| **CLIP 2** | `ltx-2.3_text_projection_bf16.safetensors` | `DualCLIPLoader` (type=ltxv) |
| **Video VAE** | `ltx-2.3-22b-dev_video_vae.safetensors` | `VAELoaderKJ` |
| **Audio VAE** | `ltx-2.3-22b-dev_audio_vae.safetensors` | `VAELoaderKJ` |
| **LoRA** | `ltx-2.3-22b-distilled-lora-384.safetensors` | strength=0.6 |
| **Camera LoRA** (可选) | `ltx-2-19b-lora-camera-control-{shot_type}.safetensors` | strength=1.0 |

### 关键 I2V 节点

```python
# 图片初始化：用 LTXVImgToVideoInplace 把首帧 baking 进 latent
video_latent = g.node("LTXVImgToVideoInplace",
    vae=video_vae[0], image=prep[0],
    latent=base_latent[0], strength=1.0, bypass=False)

# 可选 second pass：放大 2x 重编码后低噪声 refine
```

### 相比 Wan 的优势
- **GGUF 量化**：22B 模型用 Q4_K_M 量化，VRAM 需求低
- **Audio VAE 内置**：原生支持音视频生成
- **Camera LoRA**：可选控制镜头运动类型

### 模型下载
需要通过 torrent 下载（repo 里有 `ltx23.torrent`），ComfyUI 已装 `ComfyUI-LTXVideo` custom node。
